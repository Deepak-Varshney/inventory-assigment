# Inventory Management System — Backend

## Stack
- Express 5 + TypeScript
- Prisma 7 + MySQL (Aiven)
- JWT Auth + bcrypt
- Zod validation

## Setup

```bash
npm install
npm run prisma:generate
npm run prisma:push
npm run dev
```

## Folder Structure

```
src/
  controllers/    ← business logic
  routes/         ← route definitions
  middleware/     ← auth + role guard
  utils/          ← prisma client, jwt helpers
prisma/
  schema.prisma
prisma.config.ts  ← Prisma 7 config (DB URL lives here)
generated/        ← auto-generated by prisma generate (gitignored)
```

## Business Rules
- Stock can never go negative — checked before transaction
- Every stock change (add, order, cancel) creates a StockMovement record
- Order placement uses a DB transaction — order + stock deduction + history are atomic
- Order cancellation uses a DB transaction — status update + stock restore + history are atomic
- SKU is unique
- Users can only see/cancel their own orders; admins see everything

## Auth
Send `Authorization: Bearer <token>` header on all protected routes.

Register with `role: "ADMIN"` to get admin access (or set it directly in DB).
